🔍 SYNTHÈSE EXÉCUTIVE
Ce cahier des charges définit un système intelligent de nutrition sportive à double optimisation (performance athlétique + efficience économique), reposant sur :

Un moteur Python d'optimisation mathématique (PuLP / OR-Tools)
Une interface utilisateur Excel 100% paramétrable
Des bases de données alimentaires reconnues scientifiquement (CIQUAL, USDA, Open Food Facts)
Une architecture modulaire de 7 composants interconnectés

💰 MODULE 2 — BASE DE DONNÉES PRIX DYNAMIQUE


Structure de la table prix
aliment_id | magasin | prix_actuel | prix_moyen | historique | date_maj | promo_bool
Coq
Formule de score promotionnel
promo_score = (prix_moyen - prix_actuel) / prix_moyen
TOML, also INI
Un promo_score élevé → aliment en promotion → priorité d'intégration dans les repas

Fonctionnalités clés
✅ Détection automatique des opportunités d'achat
✅ Priorisation dynamique des aliments selon le rapport qualité/prix du moment
✅ Historique de prix (détection de tendances)
✅ Multi-enseignes : Carrefour, Leclerc, Lidl, Aldi, etc.

🏋️ MODULE 3 BIS — STRUCTURATION DES SÉANCES D'ENTRAÎNEMENT


Matrice des séances et impacts nutritionnels
Type de séance	Protéines	Glucides	Digestibilité avant	Priorité récupération
Musculation (force/hypertrophie)	↑↑ élevées constantes	Élevés post-séance	Medium	Musculaire
Sprint / Explosivité	Standard	↑↑ autour séance	Low (avant effort)	Nerveuse
Technique (saut à la perche)	Standard	Stables	Low-Medium	Coordination motrice
Résistance / Endurance	Standard	↑↑ max	Medium-High	Glycogène
Repos	Légèrement ↓	Réduits	High	Passive
Exemple de planning hebdomadaire
Jour	Séance	Intensité	Calories	Glucides

Mardi	Technique	🔴 Haute	Standard	Stables
Mercredi	Musculation + Sprint	🔴 Haute	+400 kcal	Très élevés
Jeudi	Repos	🟢 Faible	-200 kcal	Réduits

Samedi	Récupération active	🟡 Moyenne	Standard	Modérés
Dimanche	Repos complet	🟢 Faible	-200 kcal	Faibles


⚙️ MODULE 4 — MOTEUR D'OPTIMISATION


Fonction objectif (multi-critères)
Minimiser :
    (coût total des aliments)
  + (pénalité de répétition alimentaire)
  - (score nutrition-performance global)
  - (score promotionnel pondéré)
Python
Contraintes du modèle
# Énergie
calories_totales == cible_dynamique_journalière

# Macronutriments (basés sur les recommandations ISSN 2023)
proteines >= 1.6 * poids_kg  # minimum sportif (g/jour)
proteines <= 2.5 * poids_kg  # plafond protéique
lipides   >= 0.8 * poids_kg  # minimum hormonal
glucides  == periodises(type_seance)  # variable selon entraînement

# Budget
cout_journalier <= budget_max_journalier

# Diversité
nb_occurrences(aliment_i) <= max_repetition_semaine

# Digestibilité
digestibilite_avant_effort in ['low', 'medium']  # jamais 'high' avant effort intense

# Exclusions utilisateur
aliment not in liste_aliments_interdits
Python
Recommandations scientifiques intégrées
📚 ISSN Position Stand : 1.4–2.0 g/kg/j pour la masse musculaire ; jusqu'à 2.5 g/kg/j en phase de recomposition 📚 Burke & Jeukendrup : Périodisation glucidique "Train High / Train Low" selon charge d'entraînement 📚 Areta et al. : Distribution protéique toutes les 3–4h pour maximiser la synthèse protéique musculaire (MPS)


🏅 MODULE 5 — SCORE ALIMENTAIRE


Formule de scoring
score_aliment =
    (score_proteines × pondération_sport)
  + (score_micronutriments × pondération_santé)
  + (score_digestibilité_adaptée)
  - (coût_normalisé)
  + (promo_score × bonus_promo)
Lisp
Exemple de calibration pour le saut à la perche
Critère	Pondération
Densité protéique	30%
Digestibilité avant effort	25%
Coût par gramme de protéines	20%
Micronutriments clés (Fe, Mg, Vit D)	15%
Score promotionnel	10%
